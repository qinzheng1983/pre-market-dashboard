# 财资日报生成技能（v4.1）

## 核心原则

- **数据日期**：T（当日完整收盘数据），16:00发送
- **阅读时间**：15-20分钟（3000-5000字）
- **内容深度**：三层（L1数据层+L4洞察层+L5机构观点层）
- **格式**：HTML，CSS兼容139邮箱，表格≤5列
- **质量**：五星评级，每个数字标注来源+发布时间，关键数据双源交叉验证

## 执行前必读（强制）

1. **公司档案**：执行前读取 `COMPANY.md`，获取敞口结构、预算基准、对冲策略
2. **用户偏好**：执行前读取 `USER.md`，确认发送时间、邮箱、格式偏好
3. **模板规范**：执行前读取 `templates/finance_daily_v4_spec.md`，确认v4.1展示规范
4. **数据源铁律**：USD/CNY中间价 ONLY认央行授权公告，绝对禁止chl.cn等聚合站

## 工作流程

### Step 1: 时间验证与状态检查
```bash
date "+%Y-%m-%d %H:%M:%S %A %Z"
```
确认当前日期（YYYY-MM-DD）和星期。若为周末则跳过。
检查 `reports/finance_daily_YYYYMMDD.html` 是否已存在，避免重复生成。

### Step 2: 加载企业档案
读取以下文件（按顺序）：
1. `USER.md` — 岗位、报告偏好、预算基准
2. `COMPANY.md` — 敞口结构（Long/Short USD、镍/碳酸锂年用量、对冲策略）
3. `templates/finance_daily_v4_spec.md` — v4.1格式规范
4. `knowledge/finance_fundamentals.md` — 敞口方向检查清单（如有）

**中间价校验规则（强制执行）**：
1. 搜索"YYYY年MM月DD日 中国外汇交易中心 人民币汇率中间价公告"
2. 确认公告日期 = 报告日期
3. 提取中间价后，用 chinamoney.com.cn/bkccpr/ 二次确认
4. 离岸价必须用Investing.com + 东方财富/新浪财经双源，价差>50点需第三源验证

### Step 4: 交叉验证
- 镍：至少2源（生意社+SMM），差异>$50需第三源
- 碳酸锂：至少2源（SMM+Mysteel）
- 汇率：中间价+在岸+离岸三方对照
- 每个数字标注：来源+发布时间

### Step 5: 报告生成（v4.1 10大板块）

**10大板块顺序（绝对不可变）**：

1. **核心风险仪表盘** — 6项固定（镍/碳酸锂/汇率/地缘/大宗/央行），红黄绿灯+监控阈值
2. **对我司影响与行动** — ACTION分组标题行（深蓝背景），5列≤，预算对比，利润影响量化
3. **货币政策** — 央行操作数据+政策意图洞察+机构降准降息预测
4. **汇率动态** — USD/CNY中间价/在岸/离岸/DXY/EUR/JPY，含"对我司影响"子段落
5. **新能源** — 碳酸锂/电池价格+供需转折点分析+券商周期判断
6. **有色金属** — 铜铝镍锌铅（锡已剔除）+分品种逻辑分析+投行展望
7. **贵金属** — 黄金/白银+三重引擎逻辑+机构目标价汇总
8. **地缘风险** — 事件时间线+情景分析（基准/升级/缓和）+机构评级
9. **对冲建议（CFO操作视角）** — 6大维度表格+期权工具优先级排序
10. **数据来源与免责声明** — 时效声明表+预算基准+质量评级

**CSS红线（强制执行）**：
- ❌ `display: grid` / `display: flex`
- ❌ `linear-gradient`
- ❌ emoji字符
- ✅ `display: table`（标准HTML表格）
- ✅ 纯色背景

**表格规范**：
- 绝对禁止超过5列
- ACTION为分组标题行（`action-group`），禁止独立列
- 趋势指示器挂在数据下方：`↳ vs周+X% | vs月+Y% | vs季度+Z%`，11px，#5d6d7e
- 预算标签：绿=优于/红=超支/灰=无预算

### Step 6: 质量检查清单（生成后逐项确认）

- [ ] 执行 `date` 确认真实日期
- [ ] USD/CNY中间价是否为当日9:15后数据
- [ ] 镍/碳酸锂是否交叉验证（至少2源）
- [ ] 每个数字标注来源+时间
- [ ] 表格≤5列
- [ ] ACTION为分组行而非独立列
- [ ] "触发条件"已改为"监控阈值"
- [ ] 客户集中度/技术路线/套保执行未出现在日报
- [ ] CSS无grid/flex/gradient/emoji
- [ ] 预算对比计算已包含利润影响
- [ ] 10大板块顺序正确
- [ ] 敞口方向未搞混（负债=Short，资产=Long）

### Step 7: 保存报告
文件路径：`reports/finance_daily_YYYYMMDD.html`

### Step 8: 发送邮件
使用 `email_reporter` 工具发送至 `13911658378@139.com`：
- 主题：`财资日报 — YYYY年M月D日（周X）HH:MM CFO结构版 v4.1`
- 内容：HTML全文
- 注意：139邮箱偶有SMTP超时，首次失败后间隔5-10秒重试

### Step 9: 部署到GitHub Pages
```bash
python3 scripts/deploy_to_github.py finance-daily YYYYMMDD "摘要"
```
更新 `dashboard.html` 导航页（仅展示最新一期）。

### Step 10: 状态记录
```bash
echo "YYYY-MM-DD HH:MM CST | 财资日报 | 已生成+已邮件+已部署" >> memory/report_missing.log
```

## 质量标准

- **五星（★★★★★）**：全部数据当日+交叉验证通过
- **四星（★★★★）**：主要数据当日，少量滞后但已标注
- **三星及以下**：需补发说明

## 常见错误防范

- ❌ 使用chl.cn/汇率网/查汇率作为中间价数据源
- ❌ 将子公司美元负债判断为Long USD（正确为Short USD）
- ❌ 财资日报写成盘前简报加长版（基因不同：回顾vs前瞻）
- ❌ 表格超过5列
- ❌ ACTION作为独立列（必须为分组标题行）
- ❌ 使用flexbox/grid/linear-gradient/emoji（139邮箱不支持）
- ❌ 未标注数据来源和时间
- ❌ "触发条件"未改为"监控阈值"
- ❌ 客户集中度/技术路线/套保执行出现在日报（已移至周报）
