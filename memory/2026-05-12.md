## GitHub Pages 部署问题排查（21:19-21:32）

### 问题
finance-daily/20260512.html 在 GitHub Pages 上返回 404，但 index.html（盘前简报）正常。

### 根因发现
仓库默认分支是 **`main`**，GitHub Pages 构建源也是 `main` 分支。但所有部署脚本和本地操作都在 **`master`** 分支上提交和推送。

**验证结果**：
- `main` 分支：finance-daily 目录只到 20260511.html，dashboard.html 是 5月6日版本
- `master` 分支：有 20260512.html，dashboard.html 有 5月12日版本
- GitHub Pages 从 `main` 构建，所以看不到 master 上的新文件

### dashboard.html 合并冲突
手动合并 `master` → `main` 时，dashboard.html 出现严重冲突：
- HEAD（main 旧版）：5月6日数据
- master（新版）：5月12日数据
- 冲突标记遍布整个文件（日期、描述、footer 三处冲突）

**解决**：保留 master 版本（5月12日最新数据），放弃 main 的 5月6日旧版。

### 教训
- 新仓库默认分支是 `main`，不是 `master`
- GitHub Pages 构建源与推送分支必须一致
- 部署前必须先确认 `git branch` 当前所在分支
- dashboard.html 作为高频修改文件，容易产生合并冲突，需要特别注意
