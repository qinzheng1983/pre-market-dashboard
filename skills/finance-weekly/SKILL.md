# 财资周报 (Finance Weekly Report) Skill

## 概述

本 Skill 用于生成并部署**财资周报**（Finance Weekly Report），覆盖周一至周日完整数据，面向跨国企业 CFO/资金负责人视角，聚焦汇率、利率、大宗商品、地缘风险与对冲操作。

周报与日报的区别：
- **财资日报**：当日（T）完整收盘数据，16:00 发送，10 大板块，3000-5000 字，L1+L4+L5 三层深度
- **财资周报**：本周一至周日汇总，周五/周六 20:00 前发送，5 大板块，周度洞察 + 推演 + 下周前瞻，1500-2500 字

## 时间定义（关键修正）

**周报覆盖时间 = 本周一至本周日（含周六、周日）**

- 周一 = 本周第一个交易日
- 周五 = 本周最后一个工作日
- 周六、周日 = 市场休市但可能有新闻事件（地缘、政策、欧佩克+会议等）
- 生成时间：周日或下周一上午（不晚于周一 9:00）

**ISO 周次计算**：
```python
import datetime
week = datetime.date(2026, 6, 7).isocalendar()[1]  # 23
year = datetime.date(2026, 6, 7).isocalendar()[0]  # 2026
```

## 数据收集：独立搜索流程（与日报隔离）

**⚠️ 核心原则：周报的数据搜索独立于日报，不共用搜索流程、不共用数据缓存、不共用验证清单。**

### 为什么必须隔离
- 日报搜索的是**当日**数据，周报需要**本周全周**数据（周一至周日）
- 日报的缓存数据可能过时或不完整，周报必须重新搜索
- 周报需要**周末数据**（周六、周日），日报搜索流程不包含周末
- 周报需要**周度趋势视角**，搜索关键词和验证逻辑与日报不同

### 独立搜索流程（每次生成周报必须执行）

#### 第一步：确认时间范围
1. 执行 `date` 或 `python3 skills/datetime-utils/datetime_utils.py` 确认当前真实日期
2. 计算本周一 = 当前日期 - (当前星期几 - 1) 天（周一=1，周日=0）
3. 计算本周日 = 本周一 + 6 天
4. 确认 ISO 周次：`datetime.date.today().isocalendar()[1]`
5. 确认年份：必须是当前真实年份（如 2026），禁止混淆 2025/2026

#### 第二步：执行独立搜索（必须重新搜索，禁止复用日报数据）

| 搜索批次 | 搜索目标 | 关键词示例 | 工具 |
|---------|---------|-----------|------|
| 批次 1 | 汇率全周 | `2026年6月 USD/CNY 中间价 周一 周二 周三 周四 周五` | kimi_search |
| 批次 2 | 央行操作 | `2026年6月 央行 逆回购 MLF 买断式逆回购` | kimi_search |
| 批次 3 | 原油 | `2026年6月 Brent 原油 周涨 周跌` | kimi_search + 金投网 |
| 批次 4 | 有色金属 | `2026年6月 LME 铜 镍 铝 价格` | kimi_search + SMM |
| 批次 5 | 贵金属 | `2026年6月 黄金 白银 价格` | kimi_search + FXStreet |
| 批次 6 | 碳酸锂 | `2026年6月 碳酸锂 价格` | kimi_search + 91金属 |
| 批次 7 | 周末新闻 | `2026年6月7日 周末 欧佩克 美伊 和谈` | kimi_search |
| 批次 8 | 地缘风险 | `2026年6月 美伊 霍尔木兹 和谈` | kimi_search + 财新 |
| 批次 9 | 印尼政策 | `2026年6月 印尼矿业大会 镍矿 配额` | kimi_search |
| 批次 10 | 宏观经济 | `2026年6月 中国进出口 CPI 美国` | kimi_search |

**⚠️ 搜索规则**：
- 每个批次必须独立搜索，禁止从日报缓存中读取
- 关键词必须包含**当前年份**（如 2026年），排除所有 2025 年结果
- 每个关键数据必须至少 2 个独立来源交叉验证
- 周末数据（周六、周日）必须单独搜索，使用日期关键词如"2026年6月7日"

#### 第三步：数据验证（周报专属验证清单）

```
□ 当前真实日期已确认（年+月+日）
□ 本周一 = XXXX-XX-XX，本周日 = XXXX-XX-XX
□ USD/CNY 中间价：周一至周五逐日数据完整（5个数据点）
□ USD/CNY 即期：周五在岸即期数据已获取
□ 美元指数：本周至少获取周一、周五数据
□ 布伦特原油：周一至周五逐日收盘价或周涨跌幅
□ 周末原油：WTI 或 Brent 周末期货数据/新闻
□ 有色金属：镍、铜、铝至少获取周一和周五数据
□ 贵金属：黄金周五收盘价 + 周末动态
□ 碳酸锂：周一至周五逐日报价或周涨跌幅
□ 央行操作：本周逆回购/MLF/买断式逆回购操作记录
□ 周末新闻：至少获取1条周末重要新闻（地缘、欧佩克、政策等）
□ IDR汇率格式：已读取中国银行原始牌价（100 IDR = X CNY），手动换算为1 CNY = X IDR
□ USD/IDR交叉验证：已用USD/CNY中间价 + 100IDR/CNY牌价计算USD/IDR
□ 禁止直接使用其他来源的CNY/IDR格式数字（曾误读为10.8451，实际应为2,681）
□ 所有数据来源已标注（来源+日期+时间戳）
□ 年份确认：所有数据均为2026年，非2025年
```

**⚠️ IDR汇率格式验证（强制）：**
1. 访问中国银行外汇牌价（https://www.boc.cn/sourcedb/whpj/）
2. 找到"印尼卢比"行，记录 **100 IDR = X CNY**（中行折算价）
3. 手动换算：1 CNY = 100 / X IDR；USD/IDR = USD/CNY / X × 100
4. 例：100 IDR = 0.0373 CNY → 1 CNY = 2,681 IDR；USD/IDR = 6.8147 / 0.0373 × 100 ≈ 18,270
5. 禁止直接引用其他来源的"CNY/IDR"数字，格式混乱，曾导致247倍误差
6. 报告中IDR数据必须标注原始牌价格式"100 IDR = X CNY（中行折算价）"

**验证失败处理**：
- 若某类数据缺失，标注"⚠️ 数据缺失"，不编造数据
- 若周末无重大事件，标注"周末无重大市场事件"
- 若数据仅单来源，标注"待交叉验证"

## 数据收集范围（周一至周日）

### 1. 汇率
- USD/CNY 中间价（每日 9:15，周一至周五）
- USD/CNY 在岸即期（每日 15:00 收盘，周一至周五）
- USD/CNY 离岸（CNH，实时）
- 美元指数 DXY（周一至周五收盘价）
- 周五/周末新闻中的人民币汇率评论（如腾讯新闻 6/7 分析文章）

### 2. 货币政策
- 央行逆回购操作（周一至周五）
- MLF/买断式逆回购（若有）
- 利率变动（LPR、逆回购利率）
- 央行官员表态（周末也可能有新闻）

### 3. 大宗商品
- **原油**：Brent（周一至周五收盘），周末若有欧佩克+新闻需标注
- **有色金属**：LME 铜、铝、镍、锌、钴、钠（周一至周五收盘）
- **贵金属**：黄金、白银（周一至周五收盘，周末期货数据可补充）
- **新能源**：碳酸锂（周一至周五报价）

### 4. 地缘风险（周末新闻尤其重要）
- 美伊和谈进展（财新周刊、央视新闻、环球资讯）
- 霍尔木兹海峡通航情况
- 中东军事动态（周末可能升级）
- 其他地区冲突（俄乌、亚太）
- 欧佩克+政策会议（常在周末）

### 5. 宏观经济
- 中国进出口数据（若有）
- 美国 CPI/非农就业（若落在本周）
- 美联储/欧洲央行官员讲话
- 印尼矿业大会、RKAB 政策、HPM 价格

### 板块 3：周度洞察层（强化版）——**核心要求：分析与推演**

此板块是周报的灵魂，必须包含以下三层：

#### 3.1 本周特征提炼（What happened）
- 用 3-5 句话概括本周最核心的市场特征
- 不是数据罗列，而是**模式识别**

#### 3.2 驱动因素拆解（Why it happened）
- 每个核心变化背后有哪些驱动力？
- 区分**短期触发因素**和**中期结构性因素**
- 标注每个驱动因素的**可信度**（高/中/低）和**可逆性**（可逆/半可逆/不可逆）

#### 3.3 推演与情景分析（What happens next）
- **基准情景**（概率最大，55%）：假设当前趋势延续，下周会怎样？
- **乐观情景**（20%）：什么条件下市场会超预期向好？
- **悲观情景**（25%）：什么条件下风险会超预期恶化？
- 每个情景标注**关键触发条件**和**对我司的直接影响**

**推演示例**：
```
基准情景（55%）：美伊和谈继续拉锯，Brent 在 92-97 美元区间波动。
→ 触发条件：双方无重大突破也无重大冲突
→ 对我司影响：能源成本维持高位，套保策略不变

乐观情景（20%）：美伊签署谅解备忘录，霍尔木兹完全开放。
→ 触发条件：协议在 6 月中旬前签署
→ 对我司影响：Brent 或回落至 90-92，可减少部分原油套保

悲观情景（25%）：谈判破裂，伊朗提升铀浓缩丰度至 95%。
→ 触发条件：6 月 3 日式相互袭击升级为系统性冲突
→ 对我司影响：Brent 重回 100+，需紧急评估能源敞口
```

### ⚠️ LME 有色金属数据核对流程（强制执行）

**为什么必须核对**：2026-06-13 周报中 LME 镍使用 $18,955（实际为 5 月 11 日数据），真实 6 月 12 日仅 $17,830，误差 $1,125。用户反馈"这是我们重点关注的"。

**核对步骤（每次生成报告前必须执行）**：

1. **打开 Westmetall 官方逐日数据表**（LME 官方数据提供商）：
   - 镍：https://www.westmetall.com/en/markdaten.php?action=table&field=LME_Ni_cash
   - 铜：https://www.westmetall.com/en/markdaten.php?action=table&field=LME_Cu_cash
   - 铝：https://www.westmetall.com/en/markdaten.php?action=table&field=LME_Al_cash

2. **确认最新一日数据**：
   - 检查表格最顶行的日期是否为**最近一个交易日**
   - 记录 3-month 收盘价（`LME Nickel 3-month`列）
   - 记录 Cash-Settlement 收盘价（`LME Nickel Cash-Settlement`列）

3. **周度数据核对**：
   - 确认周一至周五（或本周交易日）的逐日收盘价
   - 计算周初→周末的涨跌幅

4. **警报规则**：
   - 若某品种价格与上月同期接近→**触发警报**：检查是否引用了过时数据
