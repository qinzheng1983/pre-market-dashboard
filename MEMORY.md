## 🧠 财经日报信息记忆与遗忘机制（2026-03-30 用户明确要求）

### 需求背景
用户要求建立信息记忆机制：
1. **记忆功能**：对已生成的报告信息按日记忆，下次生成时检验是否已呈现
2. **遗忘功能**：超过一年的信息去除，按时间加权（越近权重越高），权重低到一定程度不再记忆

### 系统设计

#### 1. 记忆存储结构
- **主文件**: `memory/finance_daily_memory.json` - 核心记忆数据库
- **设计文档**: `memory/finance_daily_memory_design.md` - 详细技术设计
- **分类体系**:
  - 货币政策 (monetary_policy)
  - 汇率 (exchange_rate)
  - 新能源 (new_energy)
  - 有色金属 (metals)
  - 贵金属 (precious_metals)
  - 能源 (energy)
  - 地缘风险 (geopolitical)
  - 市场指数 (market_indices)

#### 2. 权重计算机制
```
权重 = 0.5^(天数差/半衰期) × 重要性系数

半衰期设置:
- critical(重大地缘/政策): 90天
- high(重要数据): 30天
- medium(常规数据): 7天
- low(次要信息): 3天

重要性系数:
- critical: 2.0
- high: 1.5
- medium: 1.0
- low: 0.7
```

#### 3. 呈现策略
| 信息状态 | 处理方式 | 说明 |
|---------|---------|------|
| 首次出现 | **完整呈现** | 标注"首次" |
| 已呈现1-2次 | **简化呈现** | 简要提及 |
| 已呈现≥3次且权重>0.1 | **对比呈现** | 只展示趋势变化 |
| 权重<0.1或>365天 | **跳过** | 视为已知信息 |

#### 4. 关键规则
- **强制保留**: critical级别地缘风险信息保留180天
- **自动遗忘**: 权重<0.01或超过365天的信息自动归档
- **遗忘阈值**: 0.01（约1%关注度）
- **最大保留**: 365天

### 信息去重流程（生成报告时执行）
```
1. 收集当日原始数据
2. 对每个信息项：
   a. 计算内容哈希
   b. 查询记忆数据库
   c. 判断：新信息/已呈现/需对比/可跳过
3. 根据权重决定呈现方式
4. 更新记忆数据库
5. 清理过期记忆（>365天）
```

### 应用效果
- **避免重复**: 相同信息不会反复呈现
- **突出变化**: 对比分析帮助发现趋势
- **信息保鲜**: 自动遗忘过期信息，保持内容新鲜
- **个性化**: 根据历史呈现次数动态调整内容详略

### 文件位置
- **记忆数据库**: `memory/finance_daily_memory.json`
- **设计文档**: `memory/finance_daily_memory_design.md`

- **对冲逻辑**：若预期人民币升值，无需对冲或可降低对冲比例；若预期美元升值，需保护贸易利润

#### ② 资本层面 — 美元正敞口（Long USD）
- **业务模式**：母公司人民币借款至海外子公司
- **记账本位币**：海外子公司记账本位币为美元
- **敞口方向**：**美元正敞口**（Long USD）
- **汇率影响**：USD/CNY升值（数值越高）→ 子公司美元负债折算减少 → **对融资层面越有利**
- **对冲逻辑**：若预期人民币贬值，利于降低子公司负债账面价值；若预期人民币升值，子公司负债账面膨胀，需对冲保护

#### ③ 敞口净效应分析
- 贸易层面（负敞口）与资本层面（正敞口）存在**部分自然对冲**
- 净敞口取决于两端的金额匹配度
- 需定期评估：贸易美元流出量 vs 借款人民币折算量
- **关键判断**：当前USD/CNY走势对"净敞口"是净有利还是净不利
