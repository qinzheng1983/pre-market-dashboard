## 邮件重复发送Bug修复（2026-06-09 19:28）

**问题**：用户收到两次财资日报邮件（2026-06-09 16:06和16:30）

**根因**：
1. `report_auto_workflow.sh` 财资日报部分**缺少邮件发送状态检查**
2. 手动创建临时 `send_email.py` 直接发送，不走状态记录机制
3. 盘前简报部分虽检查邮件状态，但**发送后不记录状态**（原用`email_sent`字段但检查逻辑不匹配）

**修复内容**：
1. `report_auto_workflow.sh` v2.0 → v2.1：
   - 盘前简报：发送后记录 `"pre_market_email": true` 状态
   - 财资日报：部署后检查 `"finance_daily_email"` 状态，未发送才执行，发送后记录状态
   - 统一状态字段：`pre_market_email` / `finance_daily_email`（与部署状态 `pre_market` / `finance_daily` 分离）
2. 删除临时脚本：`send_email.py`（已删除）

**状态文件格式**：
```json
{"pre_market": true}
{"pre_market_email": true}
{"finance_daily": true}
{"finance_daily_email": true}
```

**铁律**：禁止临时创建 `send_email.py` 等不走状态记录的脚本发送邮件。所有发送必须走 `report_auto_workflow.sh`。

**验证**：脚本已测试读取，状态检查逻辑正确。
