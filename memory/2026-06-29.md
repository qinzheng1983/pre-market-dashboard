## 2026-06-29 15:33 — GitHub Pages 部署故障排查与修正

### 问题
用户报告导航页和周报URL未正确部署：
- 导航页显示旧日期（2026-04-20）
- 财资周报URL 404

### 排查过程
1. 确认本地文件存在且最新（`pre_market_briefing_20260629.html`、`finance_weekly_2026w26.html`）
2. 检查GitHub仓库文件列表，发现文件确实在根目录下
3. 直接curl测试GitHub Pages URL，发现根目录文件返回404
4. **关键发现**：通过GitHub API查询仓库Settings，发现 `source.path = "/docs"`
5. 这意味着 GitHub Pages 是从 `main` 分支的 **`docs` 目录** 部署的

### 根因
`deploy_to_github.py` 脚本一直把文件上传到仓库**根目录**，但 Pages 只读取 `docs/` 目录下的文件：
- 根目录的 `dashboard.html` 更新 → Pages 不可见
- 根目录的 `finance-weekly/` → Pages 404
- `finance-daily/` 之前碰巧在 `docs/` 里有一份旧文件，所以能访问

### 修正措施
1. **脚本修复**：`deploy_to_github.py` 新增 `PAGES_ROOT = "docs"`，所有上传路径前加 `docs/`
2. **重新部署**：
   - `docs/dashboard.html`（正确导航页）
   - `docs/index.html`（最新盘前简报 2026-06-29）
   - `docs/finance-weekly/2026w26.html`（第26周周报）
   - `docs/finance-daily/20260617.html`（6月17日日报）
3. **导航页内容**：三类报告日期和摘要全部正确

### 部署验证（2026-06-29 15:55）
| URL | 状态 | 内容 |
|------|------|------|
| `dashboard.html` | ✅ 正常 | 导航页，三卡片：6月29日盘前 / 6月17日日报 / 第26周周报 |
| `index.html` | ✅ 正常 | 最新盘前简报（原油跌破72，USD/CNY 6.8166） |
| `finance-weekly/2026w26.html` | ✅ 正常 | 第26周财资周报 |
| `finance-daily/20260617.html` | ✅ 正常 | 6月17日财资日报 |

### 教训
- 部署前必须先确认 GitHub Pages 的 source 配置（Settings → Pages → Source）
- 不能假设文件上传到仓库根目录就能被 Pages 访问
- `docs/` 目录部署模式下，所有文件必须上传到 `docs/` 路径下
