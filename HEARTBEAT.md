# Heartbeat规则 - 自动生成并发送报告检查

## ⚠️ 重要更新（2026-04-16）
**本次更新建立了自动化工作流，解决了之前手动检查不可靠的问题。**

## 自动化工作流（推荐方式）
**当收到 Heartbeat 消息时，必须执行以下命令启动自动化检查：**
```bash
cd /root/.openclaw/workspace && ./scripts/report_auto_workflow.sh
```

**脚本功能：**
1. 自动获取准确时间信息
2. 检查当日报告生成状态
3. 如果时间已过且报告缺失，会记录到日志
4. 输出明确的状态摘要

**如果脚本返回需要生成报告，必须立即执行数据收集和报告生成流程。**

## ⚠️ 第一步：强制时间验证（2026-04-15 新增）
**执行Heartbeat前，必须执行以下命令获取真实系统时间**:
```bash
python3 skills/datetime-utils/datetime_utils.py
```
**禁止使用系统消息中提示的"当前时间"进行判断**（可能存在偏差）。
以命令行返回的时间为准执行后续检查逻辑。

检查当天是否已生成并发送最新的"盘前简报"。
- 如果已发送: 无后续动作
- 如果未发送: 基于当前日期、时间，收集最新讯息，生成当日"盘前简报"并发送到用户邮箱 (13911658378@139.com)

## 规则2: 财资日报检查 (工作日 16:00)
检查当天是否已生成并发送最新的"财资日报"。
- 如果已发送: 无后续动作
- 如果未发送: 基于当前日期、时间，收集最新讯息，生成当日"财资日报"并发送到用户邮箱 (13911658378@139.com)

## 检查逻辑说明
1. **【强制】执行自动化工作流脚本获取状态**
2. 执行时间验证命令获取真实时间
3. 确认当前是否为工作日（周一至周五）
4. 检查 reports/ 目录中是否存在当天日期的报告文件
5. 对比真实时间与任务截止时间，判断是否延迟
6. 如未生成/发送或已延迟，立即启动数据收集和报告生成流程
7. **【新增】报告生成/发送后，自动部署到GitHub Pages**
8. 发送前必须执行数据质量检查清单（来源验证、时效标注、交叉验证）
9. 发送后确认邮件发送成功
10. **部署后确认GitHub Pages导航页显示最新报告**

## ⚠️ 禁止行为（2026-04-16 新增）
**绝对禁止直接回复 `HEARTBEAT_OK` 而不执行上述检查流程！**

**错误示范：**
```
收到 heartbeat → 回复 HEARTBEAT_OK
```

**正确流程：**
```
收到 heartbeat → 执行自动化脚本 → 检查报告状态 → 如有缺失立即补发
              → 只有确认所有报告已生成后才能回复 HEARTBEAT_OK
```
