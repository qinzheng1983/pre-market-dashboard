# 外汇期权策略手册

> 创建日期：2026-04-06  
> 用途：跨国企业外汇风险管理策略参考  
> 适用货币对：发达市场（EUR/USD）及新兴市场（USD/CNY等）

---

## 一、Risk Reversal（风险逆转/零成本期权）

### 策略结构
```
看涨场景：Long Call + Short Put（执行价不同）
看跌场景：Long Put + Short Call（执行价不同）
```

### 核心特征
| 维度 | 说明 |
|------|------|
| **成本** | 接近零（权利金互相抵消） |
| **收益** | 理论上无限（跟随标的走势） |
| **风险** | 下行无限（被行权的Put/Call义务） |
| **保证金** | 需要缴纳（因卖出期权） |

### 适用场景

| 市场状态 | 具体条件 | 操作建议 |
|---------|---------|---------|
| **看涨但怕跌** | 预期升值，但想降低成本 | Long Call（ATM）+ Short Put（OTM） |
| **看跌但怕涨** | 预期贬值，但想降低成本 | Long Put（ATM）+ Short Call（OTM） |
| **高波动+有方向** | IV高，单买期权太贵 | 用RR替代单买，用卖出端补贴成本 |
| **突破前夜** | 低波动，即将选择方向 | 轻仓位RR押注波动率回升+方向 |

### 市场情绪判读
- **25 Delta RR > 0**：Call比Put贵 → 市场看涨情绪占优
- **25 Delta RR < 0**：Put比Call贵 → 市场看跌情绪占优
- **RR ≈ 0**（如EUR/USD）：市场情绪均衡，波动率微笑对称

### 发达市场 vs 新兴市场
| 特征 | 发达市场（EUR/USD） | 新兴市场（USD/CNY） |
|------|-------------------|-------------------|
| 波动率微笑 | 通常对称（RR≈0） | 通常偏斜（单向Protection需求大） |
| 偏斜原因 | 央行干预双向、做市商平衡 | 贬值/升值预期集中 |
| Put持仓解读 | 可能是保护性对冲，非投机看空 | 更可能反映真实贬值担忧 |

### 风险提示
- ⚠️ 卖出期权承担无限风险，需严格止损
- ⚠️ 需要缴纳保证金，占用授信额度
- ⚠️ 临近到期时Gamma风险激增

## 二、Seagull（海鸥期权）

### 策略结构
```
基础版（看涨）：
Long Call（ATM）+ Short Put（OTM）+ Short Call（Far OTM）

基础版（看跌）：
Long Put（ATM）+ Short Call（OTM）+ Short Put（Far OTM）
```

### 盈亏特征
```
价格区间              盈亏状态
─────────────────────────────────────────
Far OTM Call以上      收益封顶（被行权）
ATM Call ~ Far Call   线性收益（核心盈利区）
ATM Call附近          最大盈利
ATM Call以下 ~ OTM Put 盈亏平衡
OTM Put以下           亏损（承担卖出Put义务）
```

### 适用场景

| 场景 | 策略意图 | 市场条件 |
|------|---------|---------|
| **温和看涨** | 赚升值收益，但设上限 | 预期小幅升值，不赌大涨 |
| **降低保险费** | 比单买Call更便宜 | 波动率中等，愿意牺牲部分上行 |
| **区间交易** | 在区间内获利 | 认为汇率在区间内波动 |

### 变种结构

#### 1. 零成本海鸥（Zero-Cost Seagull）
- 调整执行价，使净权利金为零
- 牺牲更多上行空间换取零成本

#### 2. 反转海鸥（Reverse Seagull）
- 用于表达强烈方向观点
- 结构相反：Short ATM + Long Wings

### 与其他策略对比

| 策略 | 成本 | 上行收益 | 下行风险 | 适用场景 |
|------|------|---------|---------|---------|
| 单买Call | 高 | 无限 | 有限（权利金） | 强烈看涨 |
| Risk Reversal | 低/零 | 无限 | 无限 | 高确信度看涨 |
| Seagull | 低/零 | 封顶 | 无限 | 温和看涨，有目标价 |

### 实务要点
- ✅ 适合有明确目标汇率的企业（如6.8是心理关口）
- ✅ 可将Far OTM Call设在目标止盈位
- ⚠️ 卖出Put仍承担无限风险，需设置止损
- ⚠️ 涉及3个执行价，需注意流动性和买卖价差

## 三、Butterfly（蝴蝶期权）

### 策略结构（Long Butterfly）
```
看涨蝴蝶：
Buy 1 Call (Low K) + Sell 2 Calls (Mid K) + Buy 1 Call (High K)

看跌蝴蝶：
Buy 1 Put (High K) + Sell 2 Puts (Mid K) + Buy 1 Put (Low K)

 Iron Butterfly（铁蝴蝶）：
Sell 1 Straddle + Buy 1 Strangle（更常用）
```

### 核心特征
| 维度 | 说明 |
|------|------|
| **成本** | 低（净支付权利金） |
| **最大收益** | 封顶（发生在中间执行价） |
| **最大亏损** | 有限（支付的净权利金） |
| **收益形态** | 倒V型（中间最高，两端归零） |

### 适用场景

| 场景 | 市场预期 | 策略意图 |
|------|---------|---------|
| **低波动交易** | 认为汇率将横盘 | 赚时间价值衰减 |
| **事件后平静** | 重大事件后波动率下降 | 做空波动率（Short Vega） |
| **精确点位** | 认为汇率将停在特定价位 | 最大收益在中间K |
| **低成本对冲** | 比单买期权便宜 | 限定了保护范围 |

### 盈亏分析（以Long Call Butterfly为例）

假设：USD/CNY = 7.00
- Buy 1 Call @ 6.95（成本0.08）
- Sell 2 Calls @ 7.00（收入0.10）
- Buy 1 Call @ 7.05（成本0.03）
- **净成本 = 0.01**（100 pips）

```
汇率水平      盈亏
─────────────────────
< 6.95       -0.01（最大亏损）
6.96         -0.00（盈亏平衡）
7.00         +0.04（最大盈利）
7.04         -0.00（盈亏平衡）
> 7.05       -0.01（最大亏损）
```

### Iron Butterfly（铁蝴蝶）

结构：
- Sell 1 ATM Call
- Sell 1 ATM Put  
- Buy 1 OTM Call（保护）
- Buy 1 OTM Put（保护）

特点：
- 收取净权利金（Credit Spread）
- 最大收益：汇率维持在中间价位
- 最大亏损：汇率突破两翼

### 策略对比

| 策略 | 波动率观点 | 方向观点 | 风险收益比 | 适用场景 |
|------|-----------|---------|-----------|---------|
| Long Butterfly | 看空波动率 | 中性 | 有限/有限 | 认为汇率区间波动 |
| Short Butterfly | 看多波动率 | 中性 | 有限/有限 | 认为将突破区间 |
| Iron Butterfly | 看空波动率 | 中性偏区间 | 有限/有限 | 收取时间价值 |

### 风险管理
- ⚠️ Butterfly对时间衰减敏感（临近到期时Theta影响大）
- ⚠️ 涉及多腿交易，执行成本较高（滑点、买卖价差）
- ⚠️ 中间执行价流动性可能不足
- ✅ 适合波动率高位时建仓（IV高时权利金收入多）

## 四、策略选择决策树

```
市场预期
    │
    ├─ 有明确方向 ─────────────────┐
    │                              │
    ├─ 强烈看涨/看跌 ──→ Risk Reversal（低成本表达观点）
    │                              │
    ├─ 温和看涨/看跌 ──→ Seagull（设收益上限降低成本）
    │                              │
    └─ 无方向 ─────────────────────┘
                                   │
        ├─ 预期横盘 ──→ Butterfly（做空波动率）
        │
        └─ 预期大波动 ──→ Straddle/Strangle（做多波动率）
```

## 七、Collar（领口期权）

### 策略结构
```
Long Put（ downside protection）+ Short Call（ upside cap）
核心：买保护 + 卖收益上限，通常零成本或低成本
```

### 盈亏特征
```
汇率水平              盈亏状态
─────────────────────────────────────────
低于 Put K            保护生效（最大亏损锁定）
Put K ~ Spot          随现货波动（线性）
Spot ~ Call K         随现货波动（线性）
高于 Call K           收益封顶（被行权）
```

### USD/CNY 企业套保实务

| 场景 | 企业需求 | Collar 设计 |
|------|---------|-------------|
| **进口付汇** | 怕人民币升值（买美元变贵） | Long USD Call + Short USD Put |
| **出口收汇** | 怕人民币贬值（美元收入变少） | Long USD Put + Short USD Call |
| **零成本诉求** | 不愿支付期权费 | 调整Call/Put执行价使权利金抵消 |
| **部分参与** | 保留部分上行收益 | 使用Ratio Collar（非1:1） |

### Collar vs 远期
| 对比项 | Collar | 远期 |
|--------|--------|------|
| **成本** | 零/低 | 零（但损失灵活性） |
| **下行保护** | 完全保护（到Put K） | 完全保护（锁定汇率） |
| **上行收益** | 保留到Call K | 完全放弃 |
| **灵活性** | 有（区间内浮动） | 无（必须履约） |
| **会计处理** | 可能适用套期会计 | 简单 |

### USD/CNY 特殊考量
- **中间价机制**：Collar 需考虑日间波动（中间价 vs 市场价差异）
- **区间管理预期**：如果预期央行维持区间，Collar 的 wing 可设窄一些
- **流动性**：离岸（CNH）期权流动性优于在岸（CNY），价差更小

### 变种：Zero-Cost Collar
```
调整 Put 和 Call 的执行价距离：
- 若市场有偏斜（如Put贵），可能需要Call设得更远才能零成本
- USD/CNY 在贬值压力期：Put 贵，需接受更低的收益上限
- USD/CNY 在升值压力期：Call 贵，可获得较好的保护成本
```

### 风险提示
- ⚠️ 卖出 Call 义务：到期前汇率大涨需承担被行权风险
- ⚠️ 基差风险：Collar 基于CNH/CNY期货或远期，与即期有基差
- ⚠️ 展期成本：长期套保需考虑展期时的滑点

## 八、Participating Forward（参与式远期）

### 策略结构
```
Long Forward（基准敞口）+ Long Option（参与上行）
或等效于：
Long Call + Short Put（Risk Reversal）+ 调整参与率

本质：比远期多付一点权利金，保留部分方向收益
```

### 核心特征
| 维度 | 说明 |
|------|------|
| **成本** | 低于单买期权，高于远期 |
| **参与率** | 通常50%-100%（如70%参与上行） |
| **下行保护** | 完全（类似远期锁定） |
| **保证金** | 较 RR 低（银行视结构而定） |

### 盈亏分析（USD/CNY 出口商案例）

假设：当前 7.20，预期贬值但怕升值
- 参与率：70%
- 锁定汇率：7.10（比当前差100点，但保留部分收益）

```
到期汇率    普通远期@7.10    70%参与式远期
─────────────────────────────────────────
6.80        7.10（锁死）     6.80 + (7.10-6.80)*0.7 = 7.01
7.00        7.10            7.00 + (7.10-7.00)*0.7 = 7.07
7.10        7.10            7.10
7.30        7.10            7.10 + (7.30-7.10)*0.7 = 7.24
7.50        7.10            7.10 + (7.50-7.10)*0.7 = 7.38
```

### 适用场景（USD/CNY）

| 场景 | 企业状态 | 策略意图 |
|------|---------|---------|
| **方向不确定** | 有敞口，但对汇率方向无把握 | 保底 + 部分参与 |
| **好于远期** | 当前远期点不利（如升水） | 用参与率换取更好心理价位 |
| **预算汇率** | 内部预算基于特定汇率 | 锁定最差情况，好情况分享 |
| **对冲比例不足** | 不能100%对冲（政策/额度限制） | 参与式远期提供部分保护 |

### 与结构型存款结合
银行常见包装：
- **存款 + 参与式远期**：利息补贴期权费，实现零成本
- **双币存款**：实质是卖出期权的变体，需识别风险

### USD/CNY 实务要点
- **NDF vs DF**：离岸NDF参与式结构更灵活，但需考虑监管
- **参与率报价**：不同银行差异大，建议至少3家询价
- **到期选择**：避开重大政策窗口（两会、政治局会议等）

## 九、Knock-Out / Knock-In（障碍期权）

### 策略结构
```
Knock-Out（失效型）：
- 触及障碍价 → 期权失效（作废）
- 例：USD Call @ 7.20，KO @ 7.50
- 若到期前 USD/CNY 触及 7.50，Call 作废

Knock-In（生效型）：
- 触及障碍价 → 期权才生效
- 例：USD Put @ 7.00，KI @ 6.80
- 只有 USD/CNY 先触及 6.80，Put 才生效
```

### 核心特征
| 类型 | 成本 | 风险特征 | 适用场景 |
|------|------|---------|---------|
| **Knock-Out Call** | 低（有KO风险） | 大涨时失效，踏空 | 温和看涨，设天花板 |
| **Knock-Out Put** | 低 | 大跌时失效，保护消失 | 温和看跌 |
| **Knock-In Call** | 低（可能不生效） | 未触及则白付权利金 | 突破交易 |
| **Knock-In Put** | 低 | 同上 | 跌破支撑才需保护 |

### 常见结构（USD/CNY）

#### 1. Reverse Knock-Out（反向失效）
```
出口商收汇保护：
Long USD Put @ 7.00，KO @ 6.80

逻辑：
- 正常贬值到 7.00：Put 保护生效
- 若大幅升值到 6.80：Put 失效（但此时已大幅受益）
- 成本低，适合认为不会极端升值的场景
```

#### 2. Double Knock-Out（双边失效）
```
区间交易：
Long Straddle，KO @ 7.50 / 6.80

逻辑：
- 在区间内波动：赚Gamma
- 触及任一边界：失效（控制最大损失）
- 适合 USD/CNY 区间波动预期
```

### USD/CNY 特殊考量

| 风险 | 说明 | 应对 |
|------|------|------|
| **跳空风险** | 央行中间价大幅偏离预期 | 设KO价时留足缓冲（如200-300点） |
| **连续性** | 障碍监测是持续还是收盘？ | 合同中明确（通常连续监测） |
| **反向功效** | KO 时可能正需要保护 | 避免 KO 价与核心支撑/阻力太近 |

### 与 Accumulator 结合
- **KO Accumulator**：触及障碍后整个结构终止
- **Redeemable**：累积到一定程度可提前终止

## 十、Target Redemption Forward（TARF / 累积式远期）

### 策略结构
```
核心机制：
1. 每日/每周累积固定金额外汇
2. 汇率优于约定水平时：按约定价累积（赚点差）
3. 汇率差于约定水平时：按市场价累积（可能亏损）
4. 触及目标收益 → 自动终止（Knock-Out）

常见变体：
- 单边累积（仅一个方向）
- 双边累积（两个方向同时）
- 有杠杆倍数（2x、3x）
```

### 盈亏特征（以 USD/CNY 进口商为例）

假设：
- 每月需买 100万美元
- 约定汇率：7.20（优于当前 7.30）
- 杠杆：2x（跌出区间时双倍累积）
- 目标收益：50点（KO触发）

```
汇率走势              累积结果
─────────────────────────────────────────
稳定在 7.20 以上      每月按 7.20 买 100万（赚便宜）
跌破 7.20            每月按 7.20 买 200万（杠杆生效）
触及目标收益          结构终止，不再累积
```

### USD/CNY 企业使用场景

| 场景 | 适合度 | 说明 |
|------|--------|------|
| **稳定进口** | ★★★★☆ | 有持续美元需求，想用优惠汇率 |
| **出口收汇** | ★★★☆☆ | 反向结构，需注意人民币升值风险 |
| **大额敞口** | ★★☆☆☆ | 杠杆可能在单边行情放大亏损 |
| **方向不确定** | ★☆☆☆☆ | 双边TARF可考虑，但复杂度高 |

### TARF 的风险（血泪教训）

#### 1. 杠杆风险 ⚠️⚠️⚠️
```
案例：2020年4月 USD/CNY 从 7.10 跌到 7.05
- 企业做了 3x 杠杆 TARF（卖出 USD Put）
- 原本每月买 100万，变成买 300万
- 敞口瞬间放大，被迫平仓巨亏
```

#### 2. 无限亏损风险 ⚠️⚠️⚠️
```
TARF 本质：卖出障碍期权（Knock-Out Option）
- 汇率持续走差：不断以劣于市场价累积
- 没有止损上限（除非有附加条款）
- 2022年日元、2023年人民币均有案例
```

#### 3. 流动性风险
- 结构复杂，中途平仓困难
- 银行报价价差大，提前退出成本高

### USD/CNY TARF 实务要点

#### 必须确认的合同条款
| 条款 | 关键问题 | 建议 |
|------|---------|------|
| **杠杆倍数** | 1x、2x还是3x？ | 新手只做1x，熟悉后再考虑杠杆 |
| **KO条件** | 目标收益多少？ | 设置合理，别太贪婪 |
| **观察频率** | 日、周、月？ | 频率越高风险累积越快 |
| **到期处理** | 未KO怎么办？ | 明确是否展期、如何结算 |
| **最大亏损** | 有无止损条款？ | 尽量争取最大亏损上限 |

#### 替代方案对比
| 工具 | 成本 | 风险上限 | 适合场景 |
|------|------|---------|---------|

### 监管与合规（中国企业）
- **外汇管理局**：TARF 需符合实需原则，有真实贸易背景
- **国资委**：部分央企被限制使用复杂衍生品
- **信息披露**：上市公司需披露重大衍生品头寸
