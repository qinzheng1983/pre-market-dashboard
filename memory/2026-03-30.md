## 下午 1:57 - 财经日报信息记忆与遗忘机制设计

### 用户指令
"关于财经日报，我认为你应该有一个信息的记忆机制，要求：
1. 需要对已经生成的该类型报告涉及的主要讯息有记忆，按日记忆，下次再生成日报的时候，需要检验类似的信息是否已经呈现过。如果没有记忆，可以展示该讯息；如果有记忆，则无需再呈现，只做对比分析；
2. 要有遗忘机制，对于超过一年的讯息，可以进行记忆去除，尝试给历史的讯息给予一些权重，越靠近当前的日期，讯息的权重越高，权重低到一定程度，可以不再记忆，不再进行对比分析。"

### 系统设计（已创建）

#### 核心文件
- **记忆数据库**: `memory/finance_daily_memory.json`
- **设计文档**: `memory/finance_daily_memory_design.md`

#### 权重计算机制
```
权重 = 0.5^(天数差/半衰期) × 重要性系数

半衰期:
- critical(重大地缘/政策): 90天
- high(重要数据): 30天  
- medium(常规数据): 7天
- low(次要信息): 3天
```

#### 呈现策略
| 信息状态 | 处理方式 | 权重阈值 |
|---------|---------|---------|
| 首次出现 | 完整呈现 | - |
| 已呈现1-2次 | 简化呈现 | >0.1 |
| 已呈现≥3次 | 对比呈现 | >0.1 |
| 过期信息 | 跳过/遗忘 | <0.01 或 >365天 |

#### 信息分类体系
1. **货币政策** - LPR、公开市场操作、准备金率
2. **汇率** - USD/CNY中间价/市场价
3. **新能源** - 碳酸锂价格、动力电池装机、钠电池
4. **有色金属** - 铜、铝、镍、锡、钢材、铁矿石
5. **贵金属** - 黄金、白银
6. **能源** - 布伦特原油、WTI原油
7. **地缘风险** - 中东局势、贸易摩擦、制裁
8. **市场指数** - 道指、纳指、美元指数、上证、恒生

#### 关键规则
- **遗忘阈值**: 权重 < 0.01
- **最大保留**: 365天
- **强制保留**: critical级地缘风险保留180天
- **去重逻辑**: 基于内容哈希判断是否已呈现

### 应用流程
```
1. 收集当日数据
2. 对每个信息项：
   - 计算内容哈希
   - 查询记忆库
   - 根据权重决定：present/compare/skip
3. 生成差异化报告
4. 更新记忆库
5. 清理过期记忆
```

### 效果预期
- ✅ 避免重复信息反复呈现
- ✅ 突出变化趋势（对比分析）
- ✅ 自动清理过期信息
- ✅ 内容更加精炼、个性化
